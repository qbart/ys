# Rules

Use following workflow:

1. In `project` folder you have 3 subfolders: `todo`, `in progress`, `done` - they represent tasks to be done.
Before implementation, prepare a plan and emit the tasks in form of markdown files to valid folder. When status changes, move the the tasks between the folders.
2. Build project incrementally using TDD.
3. Each small increment should be followed by git commit to save the progress.
4. Start with PRD and keep iterating.
5. When you finish whole project and all specs are working for all samples and edge cases output the <promise>PROJECT_DONE</promise> and stop.
